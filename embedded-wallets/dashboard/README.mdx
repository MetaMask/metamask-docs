## Create a new project

Before implementing any Embedded Wallet SDK, developers must create a new project in the MetaMask Developer Dashboard. This process generates a Client ID required for SDK integration.

:::important

To ensure optimal security of credentials, it's recommended to maintain separate projects for different environments (such as development, staging, production).

:::

Log in to the [MetaMask Developer Dashboard](https://developer.metamask.io) and create a new project if you haven't already.

## Get the client ID

After creating a project:

1. Navigate to **Project Settings**.
2. Select the **General** tab to obtain the `Client ID`.

The project comes pre-configured with default connections, chains, and networks to facilitate faster integration with the SDKs.

## Dashboard configuration

Once your project is created, configure your integration through the following sections:

- [Project settings](./project-settings) — basic project information, credentials, and token verification
- [Allowlist settings](./allowlist) — domain and URL authorization
- [Session management](./advanced/session-management) — session duration and behavior
- [Test accounts](./advanced/test-accounts) — development testing environment
- [User details in ID token](./advanced/user-details) — JWT token user data configuration
- [Key export settings](./advanced/key-export) — private key export permissions
- [Chains and networks](./chains-and-networks) — blockchain networks and custom chain configurations
- [Custom chains](./advanced/custom-chains) — add chains in code as well as the dashboard
- [Authentication](./authentication) — login methods, social connections, and custom authentication
- [Wallet services](./wallet-services) — wallet UI and funding options
- [Customization](./customization) — branding and login flow appearance
- [Analytics](./analytics) — project usage and user engagement metrics
