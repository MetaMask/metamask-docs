Group connections enable multiple login methods to be linked to the same on-chain user identity. This means that users logging in with different authentication providers (for example, **_Google and Email Passwordless_**) can still access the **same wallet address**â€”ensuring a unified user experience.

:::note

Before setting up a group connection, developers must first configure individual login connections that are eligible.

:::

## Set up a group connection

:::success Create a group connection

To use this feature, developers use the **Group Connections** tab in the [dashboard](https://dashboard.web3auth.io).

:::

Follow these steps:

1. Visit the [dashboard](https://dashboard.web3auth.io).
2. Go to the **Group Connections** section.
3. Click on the **Create Group** button.
4. Enter your custom `Group Name`.
5. Select `1st social connection` and `2nd social connection`.
6. Click **Create Group**.

## Usage

Since the group connections details are available from the dashboard, developers don't need to pass any additional parameters to the `Web3AuthProvider`.

:::tip

Follow our [quickstart](/quickstart/?product=EMBEDDED_WALLETS&walletAggregatorOnly=NO&framework=REACT&stepIndex=0) to set up the basic flow.
