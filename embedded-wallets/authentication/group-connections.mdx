Group Connections enable multiple login methods to be linked to the same on-chain user identity. This means that users logging in with different authentication providers (e.g., **_Google and Email Passwordless_**) can still access the **same wallet address**â€”ensuring a unified user experience.

> _Before setting up a group connection, developers must first configure individual login connections that are eligible._

## Set Up a Group Connection

:::success Create Group Connection on Dashboard

To use this feature, developers must go to the `Group Connections` tab in the [Embedded Wallets Dashboard](https://dashboard.web3auth.io).

:::

Follow these steps:

1. Visit the [Embedded Wallets Dashboard](https://dashboard.web3auth.io).
2. Go to the `Group Connections` section.
3. Click on the `Create Group` button.
4. Enter your custom `Group Name`.
5. Select `1st social connection` and `2nd social connection`.
6. Click `Create Group`.

## Usage

Since, the `Group Connections` details are available from Dashboard, developers don't need to pass any additional parameters to the `Web3AuthProvider`.

> Follow our [Quick Starts](/quickstart/?product=EMBEDDED_WALLETS&walletAggregatorOnly=NO&framework=REACT&stepIndex=0) to setup the basic flow.
