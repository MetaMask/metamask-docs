The following pages document each operation exposed by the v9 SDK hooks API. All hooks require the component to be inside a `Web3AuthProvider`.

## Hook to operation mapping

If you are migrating from v8, this table maps the old class methods to their v9 hook equivalents:

| v8 method                            | v9 hook and function                                     | Page                                      |
| ------------------------------------ | -------------------------------------------------------- | ----------------------------------------- |
| `web3auth.login({ loginProvider })`  | `useWeb3AuthConnect()` → `connectTo({ authConnection })` | [Login](./login.mdx)                      |
| `web3auth.logout()`                  | `useWeb3AuthDisconnect()` → `disconnect()`               | [Logout](./logout.mdx)                    |
| `web3auth.userInfo()`                | `useWeb3AuthUser()` → `userInfo`                         | [User info](./user-info.mdx)              |
| `web3auth.privKey` / `getPrivKey()`  | `useWeb3Auth()` → `provider` (for signing)               | [Private key](./private-key.mdx)          |
| Ed25519 key (`getEd25519PrivKey()`)  | `useWeb3Auth()` → `web3Auth.signer`                      | [Ed25519 / Solana](./ed25519-key.mdx)     |
| `web3auth.launchWalletServices()`    | `useWalletUI()` → `showWalletUI()`                       | [Wallet UI](./launch-wallet-services.mdx) |
| `web3auth.request(chainConfig, ...)` | `useSignatureRequest()` → `request(method, params)`      | [Signature request](./request.mdx)        |
| `enableMFA()` on instance            | `useEnableMFA()` → `enableMFA()`                         | [Enable MFA](./enable-mfa.mdx)            |
