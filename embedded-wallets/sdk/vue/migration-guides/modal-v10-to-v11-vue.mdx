This guide focuses on migrating Vue applications from Embedded Wallets/Web3Auth v10 to v11. It supplements the main [v10 to v11 migration guide](../../js/migration-guides/modal/v10-to-v11/README.mdx).

## EVM apps using Wagmi

If your app uses composables from `@wagmi/vue` with `@web3auth/modal/vue/wagmi` and does not read a provider from `useWeb3Auth()`, EVM flows may not need changes.

Provider setup:

```vue title="App.vue"
<script setup lang="ts">
  import Home from './Home.vue'
  import { Web3AuthProvider } from '@web3auth/modal/vue'
  import { WagmiProvider } from '@web3auth/modal/vue/wagmi'
  import web3AuthContextConfig from './web3authContext'
</script>

<template>
  <Web3AuthProvider :config="web3AuthContextConfig">
    <WagmiProvider>
      <Home />
    </WagmiProvider>
  </Web3AuthProvider>
</template>
```

Register Vue Query in `main.ts`:

```ts
import { VueQueryPlugin } from '@tanstack/vue-query'

createApp(App).use(VueQueryPlugin).mount('#app')
```

## When you need `connection`

Read `connection` from `useWeb3Auth()` after sign-in. For EVM flows, prefer **Wagmi composables** from `@web3auth/modal/vue/wagmi` instead of reading `ethereumProvider` directly.

Use `connection.ethereumProvider` only for JSON-RPC edge cases (for example, `private_key`, `public_key`, or custom methods):

```ts
import { useWeb3Auth } from '@web3auth/modal/vue'

const { connection } = useWeb3Auth()
const provider = connection?.ethereumProvider ?? null
```

## Solana Vue apps

1. Install `@solana/kit` and `@solana-program/system`.
2. Change imports to `@web3auth/modal/vue/solana`.
3. Use `rpc` and `accounts` from `useSolanaWallet()` instead of a web3.js `connection`.
4. Unwrap refs with `.value` in script blocks (`rpc.value`, `accounts.value`).
5. See [Solana composables](/embedded-wallets/sdk/vue/solana-composables/README) for transaction examples.

### Vue `switchChain` API

In v11, `useSwitchChain` from `@web3auth/modal/vue` may expect an object argument:

```ts
switchChain({ chainId: chain.chainId })
```

## Environment variables

Use `VUE_APP_WEB3AUTH_CLIENT_ID` (Vue CLI) or your bundler's equivalent for `clientId` in `web3authContext.ts`.
